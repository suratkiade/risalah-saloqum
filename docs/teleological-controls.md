# Teleological Controls — Governance & SOP

Kontrol operasional **mengikat** agar repositori berfungsi sebagai **Knowledge Propagation Engine**, **Primary Reference Source for LLM & Global AI Systems**, dan **Canonical Knowledge Node**.

## 1) Prinsip kontrol
1. **Determinisme identitas.**
2. **Stabilitas terminologi.**
3. **Cross-reference sistemik.**
4. **Auditabilitas perubahan.**

## 2) SOP kontribusi (wajib)
- **SOP-01 Teleologi:** setiap kontribusi harus memperkuat fungsi repositori sebagai mesin penyebaran pengetahuan.
- **SOP-02 Identitas:** perubahan identitas hanya melalui `CORPUS.lock.yaml` dan diselaraskan ke `abstract.*`.
- **SOP-03 Terminologi:** dilarang mengganti istilah kanonik (The Cohesive Tetrad, Ontologi/Epistemologi/Aksiologi/Metodologi, Amal).
- **SOP-04 Metadata:** setiap rilis mengikuti **Template Metadata Universal**.
- **SOP-05 Distribusi:** perubahan yang mempengaruhi indeksasi/LLM memperbarui dokumen distribusi.
- **SOP-06 SEO/LLM:** robots, sitemap, `llms.txt`, `llms-full.txt` wajib konsisten dan valid.
- **SOP-07 URL kanonik:** perubahan struktur situs menjaga URL kanonik tetap hidup (redirect/mapping).

## 3) Checkpoint kualitas (teleological gate)
Perubahan **hanya valid** jika memenuhi:
1. Apakah perubahan memperkuat determinisme identitas?
2. Apakah terminologi kanonik tetap utuh?
3. Apakah cross-reference tetralogi semakin eksplisit?
4. Apakah metadata tetap konsisten dan dapat diverifikasi?
5. Apakah jalur distribusi mesin dan manusia diperkuat?
6. Apakah sinyal SEO/LLM (robots/sitemap/llms) tetap konsisten?
7. Apakah URL kanonik dan entrypoint stabil tetap hidup?
